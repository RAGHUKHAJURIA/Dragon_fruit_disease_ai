# Dragon Fruit Disease AI

AI-powered dragon fruit analysis suite for:
- disease classification (6 classes)
- lesion detection (YOLOv8)
- quality grading (4 classes)
- visual explainability (Grad-CAM + TTA)
- VQA (image + question answering)
- advisory chatbot with Gemini + deterministic fallback

The repository now contains both research/training code and a production-style Flask web app.

---

## What Is New

Compared to older versions of this project, the current codebase includes:

- ConViTX-pretrained disease model support with automatic fallback.
- Grad-CAM pipeline with test-time augmentation (TTA) support.
- VQA module with tokenizer, answer mapping, and web/API endpoints.
- Agrobot chat endpoint using Gemini API when configured, with safe rule-based fallback.
- Upload validation gate for dragon fruit images (Gemini vision + local heuristic fallback).
- YOLO dataset setup, preparation, and validation scripts.
- Edge export scripts for both disease model and VQA model (quantization + ONNX artifacts).

## Core Capabilities

1. Disease Diagnosis
- Primary model: `models/best_convitx_pretrained.pth`
- Fallback model: `models/best_convitx.pth`
- Classes: Anthracnose, Brown_Stem_Spot, Gray_Blight, Healthy, Soft_Rot, Stem_Canker
- Explainability: Grad-CAM overlays generated by `xai/gradcam.py`

2. Lesion Detection
- Model: `models/yolo_dragon_best.pt`
- Detector class: `detect_disease.py`
- Outputs: lesion boxes, class counts, severity estimate, optional annotated image

3. Quality Grading
- Model: `models/quality_convitx.pth`
- Classes loaded from `models/quality_classes.txt` when available

4. VQA (Visual Question Answering)
- Routes and APIs available for image-question inference
- Artifacts: `models/best_vqa.pth`, `models/vqa_vocab.json`, `models/best_vqa_config.json`

5. Advisory Chatbot
- Endpoint: `/api/chat`
- Uses Gemini if `GEMINI_API_KEY` is set
- Falls back to deterministic domain responses from `chatbot/advisor.py`

6. Web Application
- Flask app: `app/main.py`
- Mobile-first UI with upload, camera, treatment dashboard, detection, quality, and VQA pages

## Flask Routes and APIs

### Pages

- `GET /` - Home dashboard
- `GET /disease` - Disease diagnosis page
- `GET /quality` - Quality grading page
- `GET /detect` - YOLO lesion detection page
- `GET /camera` - Guided camera page
- `GET /camera_result` - Camera result page
- `GET /vqa` - VQA page

### Form endpoints

- `POST /predict_disease` - Disease inference + Grad-CAM + advisory
- `POST /predict_quality` - Quality inference
- `POST /predict_detect` - YOLO detection + advisory
- `POST /predict_vqa` - VQA form inference

### JSON APIs

- `POST /api/analyze` - Base64 camera analysis flow
- `POST /api/vqa` - VQA via JSON payload
- `POST /api/vqa_query` - VQA using cached visual features (`uid` based)
- `POST /api/chat` - Agrobot chat (Gemini + fallback)

## Setup

### 1. Create virtual environment

```bash
python -m venv .venv
```

### 2. Activate environment

Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
```

macOS/Linux:

```bash
source .venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure environment (optional but recommended)

Copy `.env.example` to `.env` and set:

```env
GEMINI_API_KEY=your_key_here
GEMINI_MODEL=gemini-1.5-flash
```

If `GEMINI_API_KEY` is not set, chat still works using fallback logic.

## Required Model Artifacts

Place trained weights in `models/`:

- `best_convitx_pretrained.pth` (preferred disease model)
- `best_convitx.pth` (fallback disease model)
- `quality_convitx.pth` (quality model)
- `yolo_dragon_best.pt` (YOLO lesion detector)
- `best_vqa.pth`, `vqa_vocab.json`, `best_vqa_config.json` (VQA)

The app lazy-loads models and shows errors if missing.

## Training and Data Pipelines

### Disease model training

```bash
python models/train_convitx.py
python models/train_convitx_pretrained.py
```

### Quality model training

```bash
python models/train_quality.py
```

### YOLO data preparation

```bash
python setup_yolo_folders.py
python prepare_yolo_dataset.py
python validate_yolo_dataset.py
```

### YOLO training

```bash
python train_yolo_directml.py
```

Notes:
- Script currently forces CPU on Windows due DirectML limitations in YOLOv8 training path.
- Dataset config is in `data_dragon_lesions.yaml`.

### VQA training/export

```bash
python models/train_vqa.py
python models/export_vqa_edge.py
```

### End-to-end evaluation helpers

```bash
python models/run_pipeline.py
python models/evaluate_step3.py
python models/evaluate_tta.py
```
